Principles

Built different.

Every decision in WPUI is driven by one goal: give developers exactly what they need and nothing else.

[ ]

No framework

Pure browser APIs. querySelector, addEventListener, classList. Modern browsers are already powerful enough — use them.

{ }

No jQuery

Modern browsers don't need jQuery. Every component is written in vanilla JS that runs everywhere, no polyfills needed.

→

Copy-paste friendly

Each component is fully self-contained. Copy the three files you need. No peer dependencies, no setup, no config.

♿

Accessible by default

ARIA attributes, keyboard navigation, focus management, and focus return — built in from the start, not bolted on later.

Full code access

You own every line.

There's no black box. Every component ships as plain HTML, CSS, and JS — no compiled output, no abstraction layer. Read it, modify it, strip it down, or build on top of it. It's your code the moment you paste it in.

How it works

One pattern.
Every component.

All components share the same HTML attribute pattern. Learn it once, use it everywhere.

modal.html 
<!-- Root element identifies the component -->
<div data-wpui="modal" id="contact-modal" class="wpui-modal">

  <!-- data-open: opens the modal -->
  <button data-open>Open Modal</button>

  <div class="wpui-modal__backdrop" data-backdrop>
    <div class="wpui-modal__panel" data-panel
         role="dialog" aria-modal="true">

      <!-- data-close: closes the modal -->
      <button data-close class="wpui-modal__close">&times;</button>

      <h2>Contact Us</h2>
      <p>Modal content goes here.</p>

    </div>
  </div>

</div>

<!-- Trigger from anywhere on the page -->
<button data-modal="contact-modal">Open from nav</button>
